```
## 📐 [글 구조 - Scannability 최적화]

### Heading Hierarchy (자연스러운 구조 우선)

**기본 원칙**:
- **H1**: 페이지당 정확히 1개 (글 제목)
- **H2**: 주요 주제/섹션 구분 (자연스럽게)
- **H3**: H2 내용이 복잡할 때만 세부 구분
- **H4 이하**: 사용하지 않음 (깊이 제한)

**SEO 필수 사항**:
- 포커스 키워드를 H2 중 **최소 3개 이상**에 포함
- Heading은 **논리적 순서** 유지 (H2 → H3, H3 건너뛰고 H4 X)
- 각 Heading은 **명확하고 설명적**으로 작성

**구조 예시**:
```
H1: 메인 제목
├─ H2: 도입/배경
├─ H2: 주요 가이드
│  ├─ H3: 세부 단계 1
│  └─ H3: 세부 단계 2
├─ H2: 비교/분석 (H3 없이 진행 가능)
├─ H2: 팁/실수
├─ H2: FAQ
└─ H2: 체크리스트 (선택)
중요: 개수보다 내용의 흐름이 우선. 억지로 H2/H3 늘리지 말 것.
Word Count (분량)

최소: 2,000단어
최적: 2,200-2,500단어
최대: 2,800단어 (이상은 피로도 증가)

Paragraph Length (가독성)

모바일 최적화: 1문단 = 3-4문장 이내
데스크탑 최적화: 최대 5문장
공백 활용: 문단 사이 1줄 띄우기

Visual Elements (시각 요소)

불릿 포인트: 3개 이상 항목은 리스트로
표(Table): 비교 분석 시 필수 1회 이상
체크리스트: 실행 단계는 ☐ 형식
강조: 볼드로 핵심 용어 표시
