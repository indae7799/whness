# 개발 가이드

## 프로젝트 지침

이 프로젝트는 **`지니어스.md`**에 명시된 원칙을 개발 지침으로 따릅니다.

### 핵심 원칙

1. **명확성과 구체성**
   - 모호한 코드나 주석 지양
   - 명확한 변수명과 함수명 사용
   - 구체적인 에러 메시지 작성

2. **맥락 제공**
   - 코드의 목적과 배경 설명
   - 필요한 주석과 문서화
   - 사용 예시 포함

3. **구조화된 개발**
   - 논리적인 코드 구조
   - 단계별 개발 프로세스
   - 일관된 코딩 스타일

4. **사용자 의도 유지**
   - 요구사항 정확히 이해
   - 원래 목적에 맞는 구현
   - 불필요한 복잡도 피하기

## 개발 프로세스

1. **분석** - 요구사항의 명확성과 구체성 파악
2. **식별** - 잠재적 문제점과 개선점 찾기
3. **구현** - 명확한 구조로 코드 작성
4. **설명** - 주요 결정사항 문서화
5. **검토** - 코드 품질과 가독성 확인

## 스타일 가이드

- **간결함**: 필요한 만큼만 작성
- **균형**: 적절한 수준의 상세함
- **명확함**: 기술적 정확성과 이해 가능성

---

자세한 프롬프트 최적화 원칙은 [`지니어스.md`](./지니어스.md)를 참조하세요.
